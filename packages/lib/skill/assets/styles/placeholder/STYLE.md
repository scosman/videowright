# Style: Placeholder

## When to use

Use as a temporary starting point when no other style has been chosen yet, or as a minimal baseline to customize into a project-specific look. Replace with a real style pack or a custom style once the project's visual identity is defined.

## Aesthetic rules

- Neutral white background with dark text for maximum readability.
- System font stack for body and display -- no external font dependencies.
- Single accent color (blue) for highlights, links, and call-outs.
- Generous whitespace: large margins, comfortable line height.
- Flat design: no shadows, no gradients, no rounded corners beyond subtle radii.
- Content-first: let the text and layout do the work.

## Motion vocabulary

- Fade in for element entrances (opacity 0 to 1, 400ms, ease-out).
- Subtle slide-up for sequential reveals (translateY 20px to 0, 500ms, ease-out).
- No bounce, no spring, no overshoot. Keep motion utilitarian.
- Stagger delays of 100-200ms between sequential items.

## Don'ts

- Do not add decorative elements (borders, dividers, icons) unless they serve content.
- Do not use more than one accent color.
- Do not use gradients or background images.
- Do not animate font size or letter spacing.
