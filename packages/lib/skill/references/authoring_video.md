# Authoring a Video

A video is a timeline that references segments. This reference covers how to scaffold a video, write a timeline, and manage the authoring process.

## Folder structure

Each video lives in `videos/<name>/`:

```
videos/
  2026-01-launch/
    timeline.ts         # composition: ordered segment references + meta
    README.md           # what this video is about
    PLAN.md             # agent's working memory
    voiceover/
      script.md         # generated by `videowright script`
      audio/            # rendered VO files (later phase)
    exports/            # MP4s, gitignored
```

Segments, components, transitions, and styles live at the **repo top level**, not inside a video folder. Any video can use any segment.

```
segments/
  intro/index.ts
  feature-demo/index.ts
  outro/index.ts
components/
  animated-logo.ts
transitions/
  morph-logo.ts
styles/
  tokens.css
  STYLE.md
```

## The `timeline.ts` file

A timeline is a TypeScript module that default-exports a `Timeline` object:

```ts
import type { Timeline } from 'videowright';

const timeline: Timeline = {
  meta: {
    title: 'January 2026 Product Launch',
    aspectRatio: '16:9',         // optional, falls back to config default
    resolution: [1920, 1080],    // optional, falls back to config default
    fps: 60,                     // optional, falls back to config default
  },
  segments: [
    { id: 'intro' },
    { id: 'feature-demo', transition: 'fade' },
    { id: 'feature-cards', transition: { type: 'slideLeft', duration: 300 } },
    { id: 'outro', transition: 'fade' },
  ],
};

export default timeline;
```

### Segment entries

Each entry in the `segments` array has:

| Field | Required | Description |
|---|---|---|
| `id` | Yes | Matches a folder name under `segments/` (resolves to `segments/<id>/index.ts`) |
| `transition` | No | Built-in name (`'fade'`, `'cut'`, `'slideLeft'`, `'slideRight'`, `'slideUp'`, `'slideDown'`) or a configured object `{ type: 'name', ...options }`. Default is `'cut'`. |
| `renderBeats` | No | Array of durations in ms for render mode. One entry per `waitForNext()` call in the segment's `play()`. |

### Built-in transitions

| Name | Effect |
|---|---|
| `cut` | Instantaneous swap (default) |
| `fade` | Opacity crossfade, 400ms default |
| `slideLeft` | Incoming slides in from the right, 500ms default |
| `slideRight` | Incoming slides in from the left, 500ms default |
| `slideUp` | Incoming slides in from the bottom, 500ms default |
| `slideDown` | Incoming slides in from the top, 500ms default |

Override duration with the object form: `{ type: 'fade', duration: 800 }`.

### Custom transitions

Register custom transitions in `videowright.config.ts`:

```ts
import { defineConfig } from 'videowright';

export default defineConfig({
  projectStructure: 'v1',
  defaults: { resolution: [1920, 1080], fps: 60, aspectRatio: '16:9' },
  transitions: {
    'morph-logo': () => import('./transitions/morph-logo'),
  },
});
```

Then reference by name in the timeline: `{ id: 'outro', transition: 'morph-logo' }`.

A custom transition file exports a default function:

```ts
import type { Transition } from 'videowright';

const morphLogo: Transition = async (outgoing, incoming, ctx) => {
  // ctx.direction is 'forward' or 'backward'
  // ctx.duration is the configured duration (if any)
  const duration = ctx.duration ?? 600;

  // Animate outgoing and incoming elements using WAAPI
  const outAnim = outgoing.animate(
    [{ opacity: 1 }, { opacity: 0 }],
    { duration, fill: 'forwards' },
  );
  const inAnim = incoming.animate(
    [{ opacity: 0 }, { opacity: 1 }],
    { duration, fill: 'forwards' },
  );

  await Promise.all([outAnim.finished, inAnim.finished]);
};

export default morphLogo;
```

## Voiceover-first authoring

The recommended workflow is script-first:

1. **Write the voiceover script.** Draft the full narrative in `videos/<name>/voiceover/script.md`.
2. **Scaffold segments to match.** For each section of the script, create a segment in `segments/` with the matching `voiceover` field.
3. **Build the timeline.** Wire the segments in order in `timeline.ts`.
4. **Iterate visually.** Run `npx videowright dev` and refine animations.

The `videowright script` command can also go the other direction: it reads the `voiceover` field from each segment in timeline order and generates the concatenated script.

```bash
npx videowright script                    # prints to stdout
npx videowright script --write            # writes to videos/<name>/voiceover/script.md
npx videowright script videos/foo/timeline.ts   # explicit timeline path
```

## PLAN.md workflow

`PLAN.md` is the agent's working memory for a video. Follow this protocol:

1. **Read before iterating.** Always read `PLAN.md` before making changes to a video.
2. **Append after meaningful changes.** After script revisions, design choices, or user feedback, append a dated entry.
3. **Never delete entries.** The plan is an append-only log.

### PLAN.md template

```markdown
# PLAN: <video title>

## Initial Plan
- Target audience: ...
- Key message: ...
- Segment outline: ...

## Log

### <date> - Initial scaffold
- Created timeline with N segments
- Script drafted in voiceover/script.md

### <date> - User feedback
- User wants more emphasis on feature X
- Added internal beats to feature-demo segment
```

## Scaffolding a new video

When the user asks to create a new video:

1. Ask for the video name (suggest a date prefix).
2. Create the folder structure under `videos/<name>/`.
3. If the user has a script or topic, scaffold segments to match.
4. If not, create a minimal timeline with placeholder segments.
5. Write `README.md` with a brief description.
6. Initialize `PLAN.md` with the initial plan.
7. Run `npx videowright dev` to verify it works.

## Reordering segments

Reordering = moving array entries in `timeline.ts`. Segments configure themselves (id, voiceover, animation logic). The timeline only references them by id and optionally sets a transition.

## Promoting segments

There is no "promotion" step. Segments already live in the top-level `segments/` directory from the start. Any video's timeline can reference any segment by id.

## Running the dev server

```bash
npx videowright dev                             # auto-discovers most recent video
npx videowright dev videos/2026-01-launch/timeline.ts   # explicit path
```

The dev server:
- Opens a browser with the player
- Full-reloads on any file change
- Preserves position via URL hash (e.g., `#/feature-demo/2` = segment "feature-demo", beat 2)
- Shows a HUD (press `H` to toggle) with segment info, timing, voiceover text, and keyboard shortcuts

### Keyboard shortcuts (interactive mode)

| Key | Action |
|---|---|
| `Space` / `Right Arrow` / Click | Advance (next beat or next segment) |
| `Left Arrow` | Go back |
| `R` | Restart from first segment |
| `1`-`9` | Jump to nth segment |
| `H` | Toggle HUD |
